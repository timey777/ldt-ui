# 贡献指南 / Contributing Guide

感谢你对 LDT 的关注！本项目处于早期阶段，欢迎各种形式的贡献。

Thank you for your interest in LDT! This project is in early stage and contributions of all kinds are welcome.

---

## 如何贡献 / How to Contribute

### 报告问题 / Bug Reports

- 在 [Issues](https://github.com/timey777/ldt-ui/issues) 中提交，请附上：
  - 操作系统和版本
  - 复现步骤
  - 期望行为 vs 实际行为
- Submit via [Issues](https://github.com/timey777/ldt-ui/issues), and please include:
  - OS and version
  - Steps to reproduce
  - Expected vs actual behavior

### 功能建议 / Feature Requests

- 同样通过 [Issues](https://github.com/timey777/ldt-ui/issues) 提交
- 请先搜索是否已有类似讨论
- Also submit via [Issues](https://github.com/timey777/ldt-ui/issues)
- Please search for existing discussions first

### 提交代码 / Pull Requests

1. Fork 本仓库
2. 创建特性分支：`git checkout -b feature/your-feature`
3. 提交你的修改：`git commit -m 'Add some feature'`
4. 推送到分支：`git push origin feature/your-feature`
5. 提交 Pull Request

1. Fork this repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Commit your changes: `git commit -m 'Add some feature'`
4. Push to the branch: `git push origin feature/your-feature`
5. Open a Pull Request

### 代码风格 / Code Style

- C++17 标准
- 遵循项目现有代码风格
- 提交前确保代码可编译通过
- C++17 standard
- Follow the existing code style in the project
- Ensure the code compiles before submitting

---

## 行为准则 / Code of Conduct

请保持友善和尊重。我们致力于为所有人提供不受骚扰的参与体验。

Please be kind and respectful. We are committed to providing a harassment-free experience for everyone.
